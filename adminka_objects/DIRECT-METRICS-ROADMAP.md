# Яндекс Директ: какие данные ещё стоит забирать в админку

Ниже — практичный список метрик и срезов, которые можно получать через API Яндекс Директа (в первую очередь через Reports API) и использовать для сайта риэлтера.

## 1) Что ещё можно получить из API

### Базовые performance-метрики (помимо показов/кликов/CTR/расхода)
- `AverageCpc` — средняя цена клика.
- `AvgImpressionPosition` / `AvgClickPosition` — средняя позиция показа/клика (полезно для контроля аукциона).
- `BounceRate` — доля отказов.
- `ConversionRate`, `Conversions` — конверсия и количество целей (если настроены цели/офлайн-конверсии).
- `CostPerConversion` — стоимость конверсии.
- `Revenue`, `Profit`, `ROAS` / `ROI` — окупаемость (если передавать ценность конверсии).

### Срезы (разрезы) отчётов
- По устройствам: mobile/desktop/tablet.
- По площадкам: поиск / РСЯ.
- По регионам: область/город.
- По времени: день недели, час суток.
- По типам условий показа: ключи, автотаргетинг, аудитории.
- По поисковым запросам: реальные запросы, по которым были показы/клики.

### Качество трафика
- Доля новых/возвратных пользователей (через Метрику, если связать).
- Глубина просмотра, время на сайте, достижение микроцелей.
- Post-click поведение: какие страницы входа дают лучшие лиды.

### Управленческие данные
- Статусы кампаний/групп/объявлений.
- Ограничения бюджета и динамика дневного расхода.
- Отклонённые объявления и причины модерации.

## 2) Что полезнее всего именно для сайта риэлтера

### Приоритет 1 (обязательно)
1. **Стоимость лида (CPL)** = расход / лиды.
2. **Лиды по сегментам**: покупка/продажа/аренда + бренд/небренд.
3. **Поисковые запросы**: какие запросы дают заявки, какие «сливают» бюджет.
4. **Гео-эффективность**: откуда заявки дешевле и качественнее.
5. **Устройство + время суток**: когда и с каких устройств заявки лучше.

### Приоритет 2 (очень желательно)
1. **Воронка до обращения**: просмотр объекта → клик «позвонить/написать» → отправка формы.
2. **Качество лида (offline CRM loop)**: не только заявка, но и дошёл ли до сделки.
3. **Доля брендового трафика**: бренд часто «красивый» по CTR, но не всегда растит новый спрос.

### Приоритет 3 (для оптимизации бюджета)
1. **ROAS/ROMI** по кампаниям и сегментам.
2. **Доля показов и потери из-за бюджета/ставки** (Auction insights/статистика конкуренции).
3. **LTV-ориентированная ставка** для повторных клиентов и рекомендаций.

## 3) Рекомендуемые KPI для вашей админки

- CPL по каждой кампании.
- Конверсия в лид (Lead CVR).
- Качество лидов (доля «целевых» после ручной оценки/CRM).
- Cost per qualified lead.
- Доля неэффективных запросов (расход без лидов).
- Топ-10 прибыльных запросов / Топ-10 убыточных запросов.

## 4) Минимальный следующий шаг в реализации

1. Добавить в backend отчёт по целям/конверсиям (`Conversions`, `CostPerConversion`).
2. Добавить отчёт по поисковым запросам (search terms) с расходом и лидами.
3. В UI вывести 3 карточки: `CPL`, `Leads`, `Qualified leads`.
4. Добавить таблицу «Запросы без лидов > N BYN» для минус-слов.

## 5) Важная оговорка

Чтобы метрики типа `Conversions`, `CostPerConversion`, `ROAS/ROI` были корректны,
нужно:
- корректно связать Директ с Метрикой,
- настроить цели (форма, звонок, WhatsApp/Telegram клик, просмотр контактов),
- по возможности передавать офлайн-статус лида из CRM.
