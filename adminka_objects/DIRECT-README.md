# Яндекс Директ для Админки — Документация

## 📋 Описание

Модуль интеграции с Яндекс Директ позволяет:
- Просматривать статистику рекламных кампаний
- Анализировать показатели (показы, клики, CTR, расход)
- Сравнивать производительность выбранных кампаний
- Фильтровать данные по периодам и типам устройств

## 🚀 Установка и настройка

### 1. Требования

- PHP 7.4+ с поддержкой cURL
- Активная учётная запись Яндекс.ID
- Доступ к API Яндекс Директ

### 2. Создание OAuth приложения

1. Перейдите на https://oauth.yandex.ru/client/new
2. Заполните поля:
   - **Название** (любое): "Админка рекламных кампаний"
   - **Описание**: интеграция с админкой сайта
3. В разделе "API Яндекса" выберите:
   - ✅ **Яндекс Директ API**
4. Установите **Redirect URI**:
   ```
   https://oauth.yandex.ru/verification_code
   ```
   (** ✅ ВАЖНО**: URI должен совпадать с OAuth-сценарием через код подтверждения)
5. Сохраните приложение
6. **Скопируйте:**
   - Client ID
   - Client Secret

### 3. Первый запуск

1. Откройте https://ваш-домен.by/adminka_objects/direct.html
2. Вы увидите инструкцию по подключению
3. Вставьте полученные из OAuth приложения:
   - Client ID
   - Client Secret
   - (Опционально) Логин вашего Яндекс.ID
4. Нажмите **"Сохранить учетные данные"**

### 4. Авторизация 

После сохранения учетных данных:
1. Нажмите **"Подключить"**
2. Вы будете перенаправлены на сервер Яндекса
3. Подтвердите доступ (разрешите приложению доступ к вашему кабинету)
4. Яндекс покажет код подтверждения на странице `verification_code`
5. Вставьте код в поле "Код подтверждения" и нажмите "Активировать код"

## 📊 Функциональность

### Основные блоки

#### 1. **Статус подключения**
- Показывает, подключен ли Яндекс Директ
- Выводит логин кабинета и дату последнего обновления
- Позволяет отключиться

#### 2. **Инструкция по подключению**
- Раскрывающийся блок с пошаговыми инструкциями
- Автоматически раскрывается, если нет авторизации
- Форма для ввода Client ID и Client Secret

#### 3. **Карточки статистики**
- **Общий расход** — сумма потрачено в BYN
- **Количество кликов** — всего кликов по объявлениям
- **Количество показов** — всего показов объявлений
- **Средний CTR** — средний процент кликов от показов

Каждая карточка показывает:
- Текущее значение
- Сравнение с предыдущим периодом (если доступно)

#### 4. **Фильтры**
- **Период**: 7 дней (по умолчанию), 30 дней, произвольный диапазон
- **Тип устройства**: Все, Мобильные, Десктоп
- **Выбор кампании**: Все без ограничений
- **Кнопка обновления**: Ручное обновление данных

#### 5. **Таблица кампаний**
Содержит колонки:
- **Кампания** — название
- **Показы** — количество показов
- **Клики** — количество кликов
- **CTR** — процент кликов от показов
- **Расход** — сумма потрачено

Функции таблицы:
- ✅ Сортировка по любой колонке (клик на заголовок)
- ✅ Поиск по названию кампании (интерактивный поиск)
- ✅ Выделение строк при наведении
- ✅ Адаптивность для мобильных устройств

#### 6. **Карточки аналитики**
- **💰 Самая дорогая кампания** — максимальный расход
- **⭐ Лучший CTR** — наибольший процент CTR
- **🎯 Наибольше кликов** — максимальное количество кликов
- **⚠️ Худшая эффективность** — наименьший CTR

#### 7. **Уведомления и ошибки**
- Цветные уведомления с иконками
- Автоматическое удаление после 5 секунд (для ошибок)
- Красивое оформление в стиле админки

## 🔌 API Интеграция

### Frontend — Backend

Frontend отправляет POST запросы на `/adminka_objects/api-direct.php`:

```javascript
fetch('/adminka_objects/api-direct.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    action: 'getCampaigns',
    token: '...', 
    login: '...',
    filters: {...}
  })
})
```

### Backend — Яндекс Директ API

Backend использует OAuth токен для обращения к API Яндекс Директ:

```
POST https://api.direct.yandex.com/json/v5/campaigns
Authorization: Bearer {token}
Client-Login: {login}
```

## 🔒 Безопасность

### ⚠️ Важно

- **Client Secret** хранится в LocalStorage браузера (локально на компьютере)
- Не рекомендуется использовать на общественных компьютерах
- Не делитесь Client ID и Secret с третьими лицами
- Регулярно проверяйте активные приложения в аккаунте Яндекса

### Рекомендации

1. Используйте отдельный Яндекс.ID для админки
2. Ограничьте права приложения только доступом к API Директ
3. Периодически меняйте Client Secret в OAuth приложении
4. Используйте HTTPS для передачи данных

## 🐛 Возможные ошибки

### "Ошибка при загрузке данных"
- Проверьте интернет-соединение
- Убедитесь, что токен еще действителен
- Попробуйте отключиться и подключиться заново

### "Требуется авторизация"
- Сохраните Client ID и Client Secret
- Нажмите "Подключить" для авторизации

### "Яндекс Директ API недоступен"
- Проверьте статус API на https://api.direct.yandex.com/
- На сервере должна быть установлена библиотека cURL (curl extension)

### "Invalid redirect URI"
- Убедитесь, что в OAuth приложении точно указан:
  `https://ваш-домен.by/adminka_objects/direct.html`
- Должен совпадать с протоколом (https, а не http)
