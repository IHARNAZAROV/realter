# Изменения в системе управления куками

## Резюме изменений

Файл `js/cookie-consent.js` был полностью переписан с следующими улучшениями:

### Основные изменения:

1. **Аналитика всегда включена** ✅
   - GA4, GTM и Яндекс.Метрика теперь ВСЕГДА включены
   - Несмотря на действия пользователя (отклонить, закрыть, крестик), аналитика остаётся активной
   - В куках сохраняется `analytics: true` всегда

2. **UI изменения**
   - Удалено интерактивное переключение для Analytics из пункта меню
   - Analytics теперь отображается как "всегда включено" с disabled checkbox
   - Пользователь видит: "GA4 / GTM / Яндекс.Метрика (всегда включены)"
   - Пользователь может управлять только Marketing (реклама и ретаргетинг)

3. **Улучшенный текст**
   - Добавлено уточнение в заголовке настроек: "Аналитика (GA4, Яндекс.Метрика, GTM) всегда включена. Можно управлять только маркетингом."
   - Более информативное описание в основном блоке

4. **Оптимизация кода**
   - Код полностью распакован (был обфусцирован)
   - Добавлены комментарии для лучшей читаемости
   - Используются современные функции JavaScript (стрелочные функции, const, шаблонные строки)
   - Улучшены имена переменных для ясности

## План тестирования

### Тест 1: Первый визит пользователя
1. Откройте Dev Tools → Application → Cookies
2. Удалите куку `cookieConsent` (если существует)
3. Перезагрузите страницу (Ctrl+Shift+R)
4. ✅ Должно появиться окно согласия с куками

### Тест 2: Кнопка "Принять всё"
1. Нажмите "Принять всё"
2. Проверьте куку `cookieConsent`
3. ✅ В куке должно быть: `{"necessary":true,"analytics":true,"marketing":true,...}`
4. ✅ Окно должно закрыться
5. Перезагрузите страницу
6. ✅ Окно НЕ должно появляться

### Тест 3: Кнопка "Отклонить"
1. Удалите куку `cookieConsent`
2. Перезагрузите страницу
3. Нажмите "Отклонить"
4. Проверьте куку `cookieConsent`
5. ✅ В куке должно быть: `{"necessary":true,"analytics":true,"marketing":false,...}`
6. ✅ **ВАЖНО:** analytics ВСЕГДА true!
7. ✅ Окно должно закрыться

### Тест 4: Закрытие по крестику (X)
1. Удалите куку `cookieConsent`
2. Перезагрузите страницу
3. Нажмите крестик (X) в правом углу
4. Проверьте куку `cookieConsent`
5. ✅ В куке должно быть: `{"necessary":true,"analytics":true,"marketing":false,...}`
6. ✅ **ВАЖНО:** analytics ВСЕГДА true!

### Тест 5: Страница настроек (Settings)
1. Нажмите "Настроить"
2. ✅ Должны увидеть три раздела:
   - Necessary (disabled, всегда включен)
   - Analytics (disabled, всегда включен) ← **НОВОЕ: теперь отключено для редактирования**
   - Marketing (включен/отключен, по выбору)
3. Отметьте/снимите Marketing
4. Нажмите "Сохранить"
5. Проверьте куку

### Тест 6: Яндекс.Метрика целей
1. Откройте Dev Tools → Console
2. Нажимайте разные кнопки

### Тест 7: Проверка работоспособности window.__applyCookieConsent()
1. В консоли выполните: `window.__applyCookieConsent()`
2. ✅ Должна выполниться функция без ошибок
3. ✅ В GA4 и Яндекс.Метрике должны появиться события

## Миграция

### Если у вас есть старые куки
- Старая структура: `{necessary: true, analytics: false|true, marketing: false|true}`
- Новая структура: `{necessary: true, analytics: true, marketing: false|true}`
- При обновлении страницы старые куки НЕ будут использованы, будет показано окно согласия

## Варианты использования

```javascript
  // Ваш код для загрузки GA4, GTM, Яндекс.Метрики
  console.log('✅ Куки применены');
};
```

## Важные замечания

1. **Аналитика всегда включена** - это исполняется на уровне cookie-consent.js, поэтому не может быть отключено пользователем из UI
2. **GTM, GA4 и Яндекс.Метрика** - должны загружаться ВСЕГДА на странице (можно добавить условную логику в `window.__applyCookieConsent()`)
3. **GDPR/CCPA соответствие** - убедитесь, что это соответствует вашим юридическим требованиям
4. **Куки сохраняются на 180 дней** - измените `COOKIE_DURATION_DAYS` если нужен другой период

## Возможные проблемы и решения

### Проблема: Окно согласия не появляется после очистки куков
**Решение:** Используйте Ctrl+Shift+R для жёсткой перезагрузки (очить кэш браузера)

### Проблема: Яндекс.Метрика события не отправляются
**Решение:** Убедитесь, что `ym` объект загружен на странице перед cookie-consent.js

### Проблема: window.__applyCookieConsent() не вызывается
**Решение:** Убедитесь, что функция определена в глобальной области видимости ДО нажатия кнопок согласия

## Отладка

Для отладки в консоли браузера:
```javascript
// Текущие куки:
JSON.parse(decodeURIComponent(document.cookie.split('; ').find(c => c.startsWith('cookieConsent')).split('=')[1]))

// Удалить куку:
document.cookie = 'cookieConsent=; max-age=0; path=/'

// Исправить куку (если что-то не так):
document.cookie = 'cookieConsent=' + encodeURIComponent(JSON.stringify({necessary:true,analytics:true,marketing:false,updatedAt:new Date().toISOString()})) + '; max-age=15552000; path=/; SameSite=Lax'
```
