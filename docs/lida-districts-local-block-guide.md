# Локальный контент-блок «Районы Лиды» — пошаговая реализация

Этот гайд описывает, как реализовать на сайте SEO/контентный блок для страницы города Лида с:
- интерактивной картой районов;
- карточками инфраструктуры (школы, сады, медицина, транспорт);
- динамикой цен по районам (продажа/аренда, m², тренд).

> Формат рассчитан на текущий стек проекта: статические HTML-страницы + JS-модули + MapLibre.

---

## 1) Цель блока и ожидаемый результат

### Что должен получить пользователь
1. Быстрое сравнение районов Лиды по удобству жизни.
2. Понимание, где дешевле/дороже и как меняется цена.
3. Ответ на практический вопрос «какой район подойдет под мою задачу».

### Что должен получить бизнес
1. Рост глубины просмотра (переходы в карточки объектов по району).
2. Больше лидов с локального трафика по запросам про районы Лиды.
3. Переиспользуемый шаблон, который затем можно масштабировать на другие города.

---

## 2) Архитектура блока (из чего состоит)

Рекомендуемая структура на странице:

1. **Hero секция блока**
   - Заголовок: «Районы Лиды: карта, инфраструктура и цены».
   - Подзаголовок с датой обновления данных.

2. **Интерактивная карта районов**
   - Polygon-границы районов.
   - Цветовая заливка по средней цене за m².
   - Hover/Click popup с краткой статистикой.

3. **Панель инфраструктуры**
   - Чекбоксы/фильтры: школы, детсады, поликлиники, аптеки, остановки.
   - Счетчики по каждому району.

4. **Таблица/карточки «Динамика цен»**
   - Текущая средняя цена m².
   - Изменение за 1/3/6/12 месяцев.
   - Мини-график (sparkline) по району.

5. **Блок вывода (инсайты)**
   - «Самый доступный район», «Лидер роста», «Стабильный район».

6. **CTA**
   - Кнопка: «Подобрать квартиру в этом районе».

---

## 3) Данные: что нужно подготовить заранее

Создайте отдельный каталог, например `data/lida/`, и храните там:

1. `districts.geojson` — полигоны районов Лиды.
2. `infrastructure.json` — объекты инфраструктуры с координатами и типом.
3. `prices.json` — агрегаты по ценам.
4. `meta.json` — дата обновления, источник, версия.

### Минимальная структура `prices.json`

```json
{
  "updated_at": "2026-03-31",
  "city": "Лида",
  "districts": [
    {
      "slug": "center",
      "name": "Центр",
      "sale_price_m2": 980,
      "rent_price_m2": 9.8,
      "change_1m": 1.2,
      "change_3m": 3.5,
      "change_6m": 4.8,
      "change_12m": 9.1,
      "trend": [920, 930, 935, 945, 955, 965, 980]
    }
  ]
}
```

### Ключевые требования к качеству данных
- Единый идентификатор района (`slug`) во всех файлах.
- Одинаковая система координат (WGS84, EPSG:4326).
- Период обновления: минимум 1 раз в месяц.
- Валидация: отсутствие пустых координат и отрицательных цен.

### Вариант автозаполнения из текущей базы объектов

Если у вас все объекты уже лежат в `data/objects/*.json`, можно формировать `data/lida/*` автоматически:
- фильтр по `city = "Лида"` и `type = "Квартира"`;
- цена за m² = `priceUSD / areaTotal`;
- динамика = помесячные срезы по `publishedAt`;
- инфраструктура = извлечение сущностей из поля `infrastructure` + координаты объекта.
- геометрия районов = построение polygon (например convex hull) по координатам объектов в каждом районе.

> Для прод-версии лучше заменить расчетные полигоны на официальные границы районов (GIS/OSM/муниципальные данные).

## 4) Файловая структура в проекте

Добавьте/создайте:

- `lida-districts.html` — страница или секция (если отдельная).
- `css/lida-districts.css` — стили блока.
- `js/lida-districts.js` — инициализация карты, фильтров, рендера статистики.
- `data/lida/*.json` — источники данных.

Если блок вставляется в существующую страницу города, вместо новой HTML-страницы добавьте контейнер:

```html
<section id="lida-districts" class="lida-districts"></section>
```

## 5) Пошаговая реализация

### Шаг 1. Разметка HTML

Сделайте семантический скелет:
- `<section>` для блока;
- `<div>` под карту;
- `<aside>` под фильтры инфраструктуры;
- `<div>`/`<table>` под цены и динамику;
- `<div>` под CTA.

Важно:
- Добавьте `aria-label` для контролов.
- Укажите fallback-текст при недоступности карты.

### Шаг 2. Стили и адаптив

В `css/lida-districts.css`:
- Desktop: карта + панель справа (2 колонки).
- Mobile: стек вертикально.
- Цвета районов — с учетом контраста (AA).
- Состояния hover/focus для интерактивных элементов.

### Шаг 3. Инициализация карты (MapLibre)

В `js/lida-districts.js`:
1. Создайте карту в контейнере.
2. Подключите базовый стиль.
3. Загрузите `districts.geojson` как source.
4. Добавьте fill layer с цветом по `sale_price_m2`.
5. Добавьте line layer для границ.
6. Реализуйте hover/click popup.

Логика окраски:
- Нормализуйте диапазон цен (min/max).
- Разбейте на 5 классов или используйте интерполяцию.
- Добавьте легенду «дешевле → дороже».

### Шаг 4. Подтягивание инфраструктуры

1. Загрузите `infrastructure.json`.
2. Постройте маркеры/слой точек с фильтром по типам.
3. Для каждого района посчитайте количество объектов каждого типа (point-in-polygon).
4. Обновляйте статистику в карточке района при выборе.

Оптимизация:
- Кешируйте результат агрегирования.
- На мобильных скрывайте часть слоев до zoom ≥ 12.

### Шаг 5. Рендер «Динамика цен»

1. Загрузите `prices.json`.
2. Сопоставьте район по `slug`.
3. Отрисуйте таблицу:
   - район,
   - текущая цена m²,
   - изменение 1/3/6/12 мес,
   - мини-график.
4. Добавьте сортировку: по цене и по росту.

UX-подсказки:
- Зеленый = рост, красный = снижение.
- Для отсутствующих данных выводите «н/д», а не 0.

### Шаг 6. Инсайты и тексты

Сгенерируйте короткий summary на основе данных:
- «Самый доступный район: X»;
- «Максимальный рост за 12 мес: Y (+Z%)»;
- «Район с лучшей инфраструктурой: N объектов».

Важно, чтобы это был **автоматический расчет**, а не захардкоженный текст.

### Шаг 7. SEO-слой

Добавьте:
- H2/H3 с ключами: «районы Лиды», «цены по районам Лиды», «инфраструктура районов Лиды».
- Короткие абзацы с уникальным контентом по каждому району (2–4 предложения).
- JSON-LD (`FAQPage` или `Dataset`) при наличии подходящего контента.
- Внутренние ссылки на карточки объектов с UTM-меткой района.

### Шаг 8. Аналитика

События (например, в GA4 / GTM):
- `district_map_click`;
- `district_infra_filter_change`;
- `district_price_sort`;
- `district_cta_click`.

Параметры событий:
- `district_slug`,
- `filter_type`,
- `price_segment`,
- `device_type`.

### Шаг 9. Производительность

- GeoJSON упростить (simplify) без заметной потери формы.
- Ленивая загрузка блока (инициализация, когда секция попала во viewport).
- Минимизировать размеры JSON.
- Отложенная инициализация мини-графиков.

### Шаг 10. QA-чеклист перед публикацией

1. Карта корректно открывается на desktop/mobile.
2. Все районы кликабельны, popup показывает верные значения.
3. Фильтры инфраструктуры применяются без перезагрузки.
4. Данные в таблице совпадают с `prices.json`.
5. В Lighthouse нет критичных просадок по Performance/Accessibility.
6. События аналитики приходят в тестовый контейнер.

## 6) Пример технического плана на 5 рабочих дней

- **День 1**: сбор/нормализация данных, GeoJSON районов.
- **День 2**: HTML + CSS каркас, адаптив.
- **День 3**: карта + слои районов + popup.
- **День 4**: инфраструктура + таблица динамики + инсайты.
- **День 5**: SEO-тексты, аналитика, QA, публикация.

## 7) Что можно сделать сразу в следующей итерации

1. Добавить режимы «покупка/аренда» переключателем.
2. Добавить радиус до ключевых точек (вокзал, центр, крупный ТЦ).
3. Вынести блок в переиспользуемый модуль для других городов.
4. Добавить «сравнить 2 района» в один клик.

---

## 8) Критерии готовности (Definition of Done)

Блок считается готовым, когда:
1. Все 3 источника данных (`districts`, `infrastructure`, `prices`) автоматически подгружаются.
2. Пользователь может выбрать район на карте и сразу видит инфраструктуру + динамику цены.
3. На мобильном UX не ломается, Core Web Vitals в допустимых значениях.
4. Настроены события аналитики и проверены в боевом окружении.
5. Контент индексируется, а блок дает внутренние переходы на объекты.
