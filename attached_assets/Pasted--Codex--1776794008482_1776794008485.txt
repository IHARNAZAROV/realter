# Промт для Codex: переработка внешнего вида админки

Нужно полностью переработать ТОЛЬКО визуальную часть существующей админки сайта. Весь существующий функционал, логика, работа форм, загрузка данных, обработчики, API-запросы, localStorage, переключение разделов, модальные окна и любые JS-функции должны остаться без изменений.

Задача: превратить текущую админку в современный интерфейс в стиле premium dashboard с вертикальной навигацией, стеклянным эффектом, мягкими свечениями и плавными анимациями.

## Важно

* Ничего не ломать в существующей логике.
* Не переименовывать id, data-атрибуты, name, классы, на которых завязан JS.
* Если нужно добавить новые классы для стилизации — добавляй дополнительные классы, а не заменяй существующие.
* Не изменять структуру данных и поведение вкладок.
* Все действия должны ограничиваться HTML-обёртками, CSS и декоративными анимациями.

---

## Общая визуальная концепция

Сделать админку в тёмном футуристичном стиле:

* основной фон: глубокий тёмно-синий / графитовый градиент;
* стеклянные полупрозрачные панели;
* акцентные цвета: насыщенный сине-фиолетовый и фиолетово-розовый;
* много мягкого glow-эффекта;
* современная типографика;
* скругления 18–24px;
* тонкие полупрозрачные границы;
* плавные hover- и active-анимации.

Использовать примерно такую палитру:

```css
background: linear-gradient(135deg, #0f0f1a 0%, #1a1a2e 50%, #16213e 100%);
accent-1: #6366f1;
accent-2: #8b5cf6;
accent-3: #a855f7;
text-main: rgba(255,255,255,0.92);
text-secondary: rgba(255,255,255,0.55);
border: rgba(255,255,255,0.08);
panel-bg: rgba(255,255,255,0.04);
```

---

## Структура интерфейса

Переделать интерфейс в двухколоночную структуру:

* слева фиксированная вертикальная панель навигации шириной примерно 260–300px;
* справа основной контент активной вкладки;
* навигация всегда остаётся видимой;
* контент справа занимает всё оставшееся пространство.

Пример структуры:

```html
<div class="admin-layout">
    <aside class="admin-sidebar">
        ... вертикальные вкладки ...
    </aside>

    <main class="admin-content">
        ... существующее содержимое админки ...
    </main>
</div>
```

---

## Вертикальные вкладки

Все разделы админки оформить как вертикальные кнопки/табы в левой панели.

Каждая вкладка должна содержать:

* иконку;
* заголовок;
* маленькое пояснение под заголовком;
* плавную подсветку при наведении;
* активное состояние.

### Состояние обычной вкладки

* полупрозрачный фон;
* текст чуть приглушён;
* иконка в отдельном квадратном блоке 40x40;
* лёгкая прозрачная рамка.

### Hover

При наведении:

* вкладка слегка смещается вправо на 4–6px;
* появляется мягкое фиолетовое свечение;
* фон становится чуть ярче;
* иконка светлеет.

### Active

У активной вкладки:

* яркий градиентный фон;
* тонкая вертикальная полоска слева;
* свечение вокруг;
* стрелка справа или индикатор активности;
* иконка получает яркий сине-фиолетовый фон.

Использовать примерно такие стили:

```css
background: linear-gradient(135deg, rgba(99,102,241,.15), rgba(168,85,247,.12));
box-shadow: 0 10px 30px rgba(99,102,241,.25);
transform: translateX(4px);
```

---

## Основной блок контента

Правая часть админки должна выглядеть как большая стеклянная панель:

* backdrop-filter: blur(18px);
* лёгкий внутренний свет;
* полупрозрачный фон;
* мягкая тень;
* красивое появление содержимого.

Добавить:

* заголовок текущего раздела;
* маленький badge над заголовком;
* описание раздела;
* анимацию появления при переключении вкладок.

Анимация панели:

```css
opacity: 0 -> 1;
transform: translateX(30px) -> translateX(0);
transition: all .45s cubic-bezier(.4,0,.2,1);
```

## Карточки внутри админки

Все существующие блоки, формы, списки, таблицы, статистика, поля ввода и карточки внутри админки оформить одинаково:

* полупрозрачный тёмный фон;
* border-radius: 16px;
* тонкая белая рамка;
* при hover карточка чуть поднимается вверх;
* появляется фиолетовое свечение.

```css
background: rgba(255,255,255,0.03);
border: 1px solid rgba(255,255,255,0.06);
border-radius: 16px;
transition: all .35s ease;
```

Hover:

```css
transform: translateY(-2px);
border-color: rgba(99,102,241,0.35);
box-shadow: 0 12px 32px rgba(0,0,0,.25);
```

---

## Формы и поля ввода

Все input, textarea, select, кнопки и переключатели привести к единому стилю:

### Поля ввода

* тёмный полупрозрачный фон;
* светлый текст;
* мягкая граница;
* при focus — яркая фиолетовая подсветка.

```css
background: rgba(255,255,255,0.04);
border: 1px solid rgba(255,255,255,0.08);
color: #fff;
```

При focus:

```css
border-color: #6366f1;
box-shadow: 0 0 0 4px rgba(99,102,241,.18);
```

### Кнопки

Сделать две разновидности:

1. Primary:

* яркий сине-фиолетовый градиент;
* белый текст;
* hover с увеличением свечения.

2. Secondary:

* прозрачная;
* тонкая рамка;
* hover с лёгким затемнением.

---

## Анимации

Добавить плавные анимации по всей админке:

* переключение вкладок;
* hover карточек;
* hover кнопок;
* плавное появление контента;
* анимацию статистики;
* заполнение progress bar;
* плавное включение toggle.

Для активных элементов использовать transition не менее 0.35s.

Пример:

```css
transition: all .4s cubic-bezier(.4,0,.2,1);
```

---

## Toggle и переключатели

Все существующие checkbox и переключатели заменить визуально на современные switch:

* овальная дорожка;
* круглый бегунок;
* при включении — градиентный фон;
* бегунок плавно смещается вправо.

---

## Адаптивность

На мобильной версии:

* вертикальная панель должна превращаться в горизонтальную прокручиваемую ленту;
* иконки и заголовки остаются;
* sidebar не должен ломать существующую мобильную логику;
* контент должен переноситься ниже.
