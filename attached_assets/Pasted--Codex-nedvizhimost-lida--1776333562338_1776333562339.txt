# Промт для Codex

Нужно доработать страницу `nedvizhimost-lida` и добавить в существующий блок фильтров два интерактивных range-слайдера:

1. Слайдер цены в белорусских рублях.
2. Слайдер площади объекта в м².

Оба слайдера должны быть встроены в текущий блок фильтрации максимально органично, в стиле существующего интерфейса. Внешний вид ориентировать на приложенный пример: тонкая линия, активный цвет как у текущих акцентных элементов сайта, круглый бегунок, подписи минимального и максимального значения под шкалой, сверху крупное текущее выбранное значение.

Требования:

* Не ломать существующую логику фильтрации.
* Новые фильтры должны работать одновременно с уже существующими фильтрами.
* При изменении любого из слайдеров карточки объектов должны обновляться сразу, без перезагрузки страницы.
* Фильтрация должна происходить в реальном времени.
* Если объект не подходит одновременно под выбранную цену и площадь, его карточка должна скрываться.

Источник данных:

* Все значения брать из `objects.json`.
* Для цены использовать поле стоимости объекта.
* Для площади использовать поле общей площади объекта.

Важно:

* Минимальное и максимальное значение для обоих слайдеров нельзя задавать вручную.
* При загрузке страницы нужно автоматически вычислить:

  * минимальную цену среди всех объектов;
  * максимальную цену среди всех объектов;
  * минимальную площадь;
  * максимальную площадь.
* Если позже в `objects.json` появятся новые объекты с большей ценой, меньшей ценой, большей площадью или меньшей площадью — границы слайдеров должны автоматически пересчитываться.

Логика интерфейса:

Для блока цены:

* Заголовок: `Бюджет`.
* Под заголовком отображать текущее выбранное значение:

  * если выбран максимум — выводить `Любой бюджет`;
  * иначе, например: `До 120 000 BYN`.
* Под слайдером слева показать минимальную цену, справа максимальную.
* Формат чисел: пробелы между тысячами, например `125 000 BYN`.

Для блока площади:

* Заголовок: `Площадь`.
* Под заголовком отображать:

  * если выбран максимум — `Любая площадь`;
  * иначе, например: `До 85 м²`.
* Под слайдером слева показать минимальную площадь, справа максимальную.

Размещение:

* Оба фильтра должны располагаться рядом в одной строке на десктопе.
* На мобильных устройствах блоки должны становиться друг под другом.
* Использовать адаптивную вёрстку.
* Не нарушать внешний вид существующих фильтров, карточек и блока с количеством найденных объектов.

Также обновить счётчик найденных объектов:

* После изменения любого фильтра количество найденных объектов должно пересчитываться.
* Текст вида: `44 объектов найдено`.

Технические требования:

* Использовать только vanilla JavaScript.
* Не использовать сторонние библиотеки или фреймворки.
* Код должен быть разделён на:

  * HTML-разметку фильтров;
  * CSS-стили;
  * JS-логику.
* Не переписывать существующий код полностью — только аккуратно встроить новые элементы.
* Если на странице уже есть функция фильтрации, расширить её, а не создавать дублирующую.

Пример структуры JS:

* После загрузки `objects.json`:

  * вычислить диапазоны;
  * установить `min`, `max`, `value` для обоих input[type="range"];
  * обновить текстовые подписи;
  * вызвать существующую функцию рендера карточек.

* На событие `input` каждого range:

  * обновлять отображаемое значение;
  * фильтровать массив объектов;
  * обновлять карточки и счётчик.

Стилизация:

* Использовать современный минималистичный стиль.
* Цвет активной части слайдера — фирменный бирюзово-зелёный сайта.
* Бегунок круглый, с белым центром и цветной обводкой.
* Высота трека около 4px.
* Между двумя фильтрами сделать одинаковые отступы.
* Добавить плавную анимацию изменения.

Сгенерируй готовые изменения для HTML, CSS и JavaScript с конкретными фрагментами кода и укажи, в какой файл и после какого блока их вставить.
