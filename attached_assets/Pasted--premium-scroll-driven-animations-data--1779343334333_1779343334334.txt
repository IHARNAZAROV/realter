Добавь на страницу аналитики недвижимости premium scroll-driven animations в стиле современных data visualization платформ.

Основная задача:
создать ощущение «живого рынка», где графики, данные и аналитика постепенно оживают во время скролла страницы.

Анимации должны выглядеть дорого, плавно и современно.
Вдохновение:
— Apple
— Stripe
— TradingView
— Linear
— Bloomberg dashboards

Использовать:
— vanilla JavaScript
— IntersectionObserver
— requestAnimationFrame
— CSS transforms
— GPU accelerated animations

Не использовать тяжёлые animation libraries.

========================================
ОБЩАЯ КОНЦЕПЦИЯ
===============

При скролле:
— графики постепенно рисуются
— линии тренда появляются слева направо
— цифры плавно увеличиваются
— карточки появляются с stagger delay
— элементы слегка parallax смещаются
— создаётся ощущение динамики рынка

========================================

1. DRAW ANIMATION ДЛЯ ГРАФИКОВ
   ========================================

Для Chart.js графиков реализовать:

— плавное рисование линии
— постепенное появление точек
— delayed animation datasets
— easing: cubic-bezier(.22,.61,.36,1)

Линия должна:
не появляться мгновенно,
а «прорисовываться» по мере появления блока на экране.

Анимация запускается:
только один раз при первом scroll reveal.

========================================
2. COUNT-UP АНИМАЦИИ
====================

Для всех ключевых цифр:

— средняя цена
— стоимость м²
— индекс доступности
— количество лет накоплений
— рост цен
— процент спроса

Реализовать плавный count-up.

Пример:
0 → 124 500 BYN

Требования:
— easing animation
— плавное замедление в конце
— requestAnimationFrame
— форматирование чисел через Intl.NumberFormat

========================================
3. TREND LINE REVEAL
====================

Для trend blocks:

— линия тренда появляется постепенно
— glow trail effect
— fade opacity
— motion blur illusion

Линия должна выглядеть:
как будто система анализирует рынок в realtime.

========================================
4. STAGGER ANIMATION
====================

Карточки аналитики:
— не появляются одновременно
— каждая карточка появляется с задержкой 80–120ms

Эффекты:
— opacity
— translateY
— blur reduction

========================================
5. SCROLL PARALLAX
==================

Добавить subtle parallax:

— background gradients
— glow circles
— floating indicators
— decorative grid

Движение:
очень лёгкое и премиальное.

Без агрессивных эффектов.

========================================
6. LIVE DATA EFFECT
===================

Создать ощущение realtime analytics.

Добавить:
— pulse indicators
— blinking dots
— shimmer highlights
— animated trend arrows

Пример:
🟢 Рынок активен
↑ +8.4%

========================================
7. MARKET TICKER EFFECT
=======================

Для ключевых market metrics:

— плавное изменение чисел
— мягкое обновление значения
— короткая подсветка изменения

Пример:
118 400 → 119 100

С кратким glow flash.

========================================
8. SCROLL TIMELINE ANIMATION
============================

Для historical timeline:

При скролле:
— годы появляются постепенно
— timeline line рисуется
— milestones активируются glow animation

========================================
9. HOVER INTERACTIONS
=====================

На hover:
— графики слегка усиливают glow
— карточки поднимаются
— chart points увеличиваются
— trend lines становятся ярче

========================================
10. SMART VISUAL FEEDBACK
=========================

При попадании блока в viewport:

— section softly illuminates
— появляется subtle shadow glow
— background grid становится чуть ярче

========================================
11. PERFORMANCE
===============

Критически важно:

— использовать IntersectionObserver
— passive event listeners
— will-change только где необходимо
— transform вместо top/left
— opacity + translate3d
— throttle scroll events
— минимизировать repaint/reflow

Анимации должны работать плавно даже на мобильных устройствах.

========================================
12. REDUCED MOTION SUPPORT
==========================

Добавить:

@media (prefers-reduced-motion: reduce)

В этом режиме:
— отключить parallax
— отключить draw animations
— отключить stagger
— оставить только fade

========================================
13. MOBILE UX
=============

На мобильных:
— снизить интенсивность эффектов
— уменьшить blur
— сократить distance translate
— отключить тяжёлые glow эффекты

========================================
14. CSS ARCHITECTURE
====================

Создать:

/css/scroll-animations.css
/js/scroll-animations.js

Использовать:
— CSS variables
— utility animation classes
— reusable reveal system

========================================
15. REUSABLE SYSTEM
===================

Сделать универсальные классы:

[data-animate]
[data-countup]
[data-chart-reveal]
[data-stagger]
[data-parallax]

Чтобы система легко масштабировалась на другие страницы.

========================================
16. ВИЗУАЛЬНЫЙ РЕЗУЛЬТАТ
========================

Интерфейс должен ощущаться как:

— premium market terminal
— современная аналитическая платформа
— живой рынок недвижимости
— интерактивный financial dashboard

Анимации должны усиливать доверие к аналитике,
а не отвлекать пользователя.

Главная цель:
создать эмоциональное ощущение:
«данные постоянно обновляются и рынок живёт».
