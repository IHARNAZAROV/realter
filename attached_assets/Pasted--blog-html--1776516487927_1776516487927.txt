Ты перерабатываешь страницу blog.html. Сейчас статьи выводятся длинной лентой. Нужно превратить страницу в полноцененный контентный раздел с удобной навигацией и быстрым доступом к старым статьям.

ВАЖНО:
- Внешний вид самих карточек статьи НЕ менять.
- Сохранять текущий корпоративный стиль сайта: цвета, шрифты, тени, скругления, анимации.
- Использовать только vanilla JavaScript, без библиотек и фреймворков.
- Все данные продолжать брать из blog-articles.json.
- Код должен быть аккуратно разделён: HTML структура, CSS стили и JS логика.
- Ничего не ломать в мобильной версии: на мобильных устройствах всё должно корректно перестраиваться.

Нужно сделать новую структуру страницы:

1. Общий layout:
- Контентная область делится на 2 колонки.
- Слева фиксированный sidebar шириной примерно 280px.
- Справа основной контент с карточками статей.
- Между колонками сделать отступ 40px.
- На ширине меньше 992px sidebar должен уходить вверх над контентом и становиться обычным блоком.

Структура:

-------------------------------------------------
| Sidebar                       | Статьи        |
|                               |               |
| Категории                     | 2 колонки     |
| Поиск                         | карточек      |
| Последние статьи              |               |
| Архив по месяцам              |               |
-------------------------------------------------

2. Sidebar:
Создать слева аккуратный sticky sidebar.

Sidebar должен прилипать при прокрутке:
- position: sticky
- top: 120px

Внутри sidebar должны быть 4 блока.

=== Блок 1. Категории ===

Заголовок:
"Категории"

Список категорий:
- Все статьи
- Последние статьи
- Юридические риски
- Покупка / продажа
- Психология
- Рынок недвижимости
- Наследство
- Перепланировка

Для каждой категории справа вывести количество статей в скобках.

Например:
Юридические риски (8)

Категории должны формироваться автоматически из blog-articles.json:
- использовать category и tags
- если категория отсутствует, не показывать её

Активная категория:
- фон в фирменном цвете
- белый текст
- плавная анимация

При клике:
- фильтровать карточки справа
- сбрасывать пагинацию / кнопку "показать ещё"
- плавно прокручивать страницу к началу списка карточек

=== Блок 2. Поиск ===

Ниже категорий разместить поиск по статьям.

Заголовок:
"Поиск"

Поле ввода с placeholder:
"Введите название статьи..."

Поиск должен работать мгновенно без кнопки:
- по title
- по tags
- по category

Если результатов нет:
показывать аккуратный блок:
"Ничего не найдено. Попробуйте другой запрос."

=== Блок 3. Последние статьи ===

Заголовок:
"Последние статьи"

Показывать 3 самые новые статьи из JSON.

Каждая статья в sidebar:
- миниатюра 70x70
- заголовок максимум в 2 строки
- дата
- ссылка на статью

Карточки последних статей должны быть компактными, в одну колонку, с hover-эффектом.

Важно:
не использовать те же большие карточки, что в основном списке.

=== Блок 4. Архив ===

Заголовок:
"Архив"

Сгруппировать статьи по месяцам и годам.

Пример:
2026
- Февраль (4)
- Январь (5)

2025
- Декабрь (7)
- Ноябрь (6)
- Октябрь (4)

При клике на месяц:
- справа показывать только статьи выбранного месяца
- выделять активный пункт

Использовать русские названия месяцев.

3. Основная часть справа

В основной области:
- карточки отображаются в 2 колонки.
- На больших экранах 2 колонки.
- Ниже 768px автоматически переходить в 1 колонку.

Пример:
grid-template-columns: repeat(2, minmax(0, 1fr));

Расстояние между карточками:
- 32px по горизонтали
- 40px по вертикали

Карточки должны быть одинаковой высоты в пределах строки.

4. Ограничение количества статей

Сразу показывать только первые 8 статей.

Под карточками добавить большую кнопку:

"Показать ещё статьи"

При каждом клике:
- подгружать ещё 6 статей
- без перезагрузки страницы
- плавно анимировать появление

Когда статьи закончились:
- кнопку скрывать

Важно:
при фильтрации по категории, поиску или архиву снова сначала показывать только первые 8 статей.

5. Дополнительные UX-улучшения

Над карточками справа добавить строку состояния:

Примеры:
- Найдено 34 статьи
- Юридические риски: 8 статей
- По запросу «наследство» найдено 3 статьи

Эта строка должна обновляться автоматически.

Если пользователь нажал категорию или архив, над сеткой показывать небольшую кнопку:

"Сбросить фильтр"

После нажатия:
- возвращаются все статьи
- поиск очищается
- активные пункты sidebar сбрасываются

6. Анимации

Сделать плавные анимации:
- hover у категорий
- плавное появление карточек
- анимация кнопки "Показать ещё"
- плавное изменение количества карточек

Для карточек использовать:
- opacity
- translateY(20px)
- transition 0.35s ease

7. Стили sidebar

Sidebar должен выглядеть как дорогой современный блог, а не как обычный список.

Требования:
- белый фон
- скругление 24px
- лёгкая тень
- отдельные карточки-блоки внутри sidebar
- внутренние отступы 24px
- расстояние между блоками 24px

Пример визуального стиля:
- фон: #ffffff
- border-radius: 24px
- box-shadow: 0 12px 40px rgba(0,0,0,0.08)

Для активных пунктов использовать фирменный цвет сайта.

8. Обязательно сохранить SEO и текущую структуру

Нельзя ломать:
- существующие ссылки на статьи
- текущую разметку карточек
- schema.org
- мета-теги
- существующую фильтрацию, если она уже есть

Нужно просто расширить её и переписать под новую архитектуру.

9. Реализация JavaScript

Создать отдельный JS-модуль, например:
- /js/blog-sidebar.js

В нём:
- загрузка JSON
- построение sidebar
- фильтрация
- поиск
- архив
- "показать ещё"
- обновление строки состояния

Код должен быть хорошо структурирован:

- loadArticles()
- renderSidebar()
- renderArticles()
- applyFilters()
- updateResultsInfo()
- setupLoadMore()

10. Дополнительно

Если возможно:
- сохранить выбранную категорию или поиск в URL через query params.
Пример:
blog.html?category=legal
blog.html?search=наследство

Чтобы после обновления страницы фильтр не сбрасывался.

И ещё:
- при открытии страницы, если в URL есть category, archive или search — автоматически применить этот фильтр.

Сделай полностью готовую реализацию без заглушек и без комментариев "добавьте сами". Всё должно быть сразу интегрировано в существующую страницу blog.html и работать с текущим blog-articles.json.
