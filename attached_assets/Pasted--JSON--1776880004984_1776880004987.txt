ЗАДАЧА:
Реализовать счетчик просмотров для статей блога, данные которых хранятся в JSON.

Пример структуры статей:
см. JSON файл (каждая статья содержит поля id, slug, title, content и т.д.) :contentReference[oaicite:0]{index=0}

---

КРИТИЧЕСКИ ВАЖНО:
- сайт на vanilla JavaScript (без фреймворков)
- статьи подгружаются и рендерятся из JSON
- НЕЛЬЗЯ менять структуру JSON (кроме добавления id там, где его нет)
- нельзя ломать текущий рендер
- решение должно быть масштабируемым (в будущем можно заменить на backend)

---

ЦЕЛЬ:
Добавить счетчик просмотров:
1. на странице статьи
2. в карточке статьи (список блога)

---

РЕАЛИЗАЦИЯ:

1. УНИКАЛЬНЫЙ ИДЕНТИФИКАТОР

- использовать поле "id" как основной ключ
- если у статьи отсутствует id:
  → автоматически использовать slug как fallback

Пример:
const postId = article.id || article.slug;

---

2. ХРАНЕНИЕ ДАННЫХ

Использовать localStorage:

ключ:
"blog_post_views"

структура:
{
  "free-realtor-consultation": 10,
  "cancel-gift-contract": 4
}

---

3. УВЕЛИЧЕНИЕ ПРОСМОТРОВ

При открытии страницы статьи:

- определить текущую статью (по slug или id)
- получить postId
- загрузить данные из localStorage
- если записи нет → создать = 1
- если есть → увеличить на 1

ВАЖНО:
- инкремент происходит ОДИН раз при загрузке страницы
- не должен повторяться при повторном рендере или обновлении DOM

---

4. ФУНКЦИИ

Создать отдельный модуль (например blog-views.js):

Функции:

getViews(postId)
→ возвращает число просмотров (если нет — 0)

incrementViews(postId)
→ увеличивает счетчик

getAllViews()
→ возвращает объект всех просмотров

---

5. ИНТЕГРАЦИЯ СО СТРАНИЦЕЙ СТАТЬИ

После рендера статьи:

- вызвать incrementViews(postId)
- затем получить актуальное значение
- вывести в DOM

Пример вывода:
"👁 123 просмотров"

Расположение:
под заголовком статьи (title)

---

6. ИНТЕГРАЦИЯ СО СПИСКОМ СТАТЕЙ

При рендере карточек:

- для каждой статьи:
  → получить postId
  → вызвать getViews(postId)
  → отобразить

Пример:
👁 45

---

7. UI / СТИЛИ

- использовать текущий корпоративный стиль
- минималистично
- не ломать верстку карточек
- иконка:
  → SVG или 👁

---

8. ИЗОЛЯЦИЯ КОДА

- не засорять глобальную область
- использовать IIFE или модульный подход
- все функции сгруппировать

---

9. УСТОЙЧИВОСТЬ

- предусмотреть:
  - отсутствие localStorage (fallback)
  - поврежденные данные (try/catch)

---

10. ГОТОВНОСТЬ К BACKEND

- структура должна позволять легко заменить localStorage на API

---

РЕЗУЛЬТАТ:

- при открытии статьи счетчик увеличивается
- при возврате в список отображается актуальное значение
- все работает без backend
- корректно работает даже если у статьи нет id (используется slug)
