Создай дополнительный пункт в навигационном меню главной страницы, который располагается строго после всех обычных пунктов меню.

Это не ссылка и не кнопка, а отдельный динамический текстовый блок внутри header/nav.

Требования к логике:

1. Получать данные из objects.json.
2. Найти количество объектов, которые находятся в продаже.
3. Сначала показывать текст:

   Всего в продаже N объектов

   где N — количество объектов со статусом "в продаже" или аналогичным статусом в objects.json.

4. Затем автоматически через несколько секунд текст должен плавно сменяться на:

   Последний объект добавлен M часов назад

   либо:
   Последний объект добавлен M дней назад

   в зависимости от даты последнего добавленного объекта из objects.json.

5. После этого цикл повторяется бесконечно.

Главное требование — реализовать красивый «эффект барабана» или вертикального роллера текста:
- текущая строка плавно уезжает вверх;
- следующая строка плавно появляется снизу;
- без рывков;
- без мерцания;
- без изменения ширины и высоты контейнера;
- текст не должен прыгать;
- контейнер всегда остаётся одинакового размера;
- анимация должна быть очень плавной и дорогой визуально, как у премиальных сайтов.

Технические требования к анимации:
- использовать transform: translateY(...) и opacity;
- обязательно использовать will-change: transform, opacity;
- высота контейнера фиксированная;
- overflow: hidden;
- строки должны располагаться абсолютно внутри контейнера;
- заранее резервировать максимальную ширину, чтобы при смене текста блок не менял размер;
- продолжительность анимации примерно 600–800ms;
- интервал между сменой текста примерно 4–5 секунд;
- easing использовать cubic-bezier(0.22, 1, 0.36, 1);
- при переключении не должно быть layout shift.

Требования к стилю:
- блок должен визуально выглядеть как продолжение меню;
- по высоте, вертикальному выравниванию и отступам он должен совпадать с nav-item;
- но при этом слегка выделяться:
  - чуть более насыщенный фон;
  - тонкая полупрозрачная граница;
  - лёгкое свечение или мягкая бирюзовая/синяя подсветка;
  - можно добавить маленькую точку-индикатор или мини-иконку перед текстом;
- стиль должен быть современный, дорогой, минималистичный;
- не делать слишком ярким или похожим на кнопку CTA;
- шрифт, размер и межбуквенный интервал должны совпадать с меню.

Пример желаемой структуры HTML:

<nav class="main-nav">
  ...
  <a class="nav-link">О нас</a>
  <a class="nav-link">Контакты</a>

  <div class="nav-market-status" id="marketStatus">
    <span class="status-indicator"></span>

    <div class="status-text-wrapper">
      <div class="status-text current"></div>
      <div class="status-text next"></div>
    </div>
  </div>
</nav>

Требования к JavaScript:
- написать полностью рабочий vanilla JavaScript;
- без библиотек;
- после загрузки objects.json автоматически вычислять:
  - количество объектов в продаже;
  - дату последнего объекта;
- формировать обе строки текста;
- организовать бесконечную циклическую смену строк;
- использовать requestAnimationFrame или аккуратную последовательность через classList;
- не допускать наложения нескольких таймеров;
- при повторной инициализации старый интервал должен очищаться;
- предусмотреть защиту, если objects.json пустой или не загрузился.

Если данные не получены, показывать текст:

Нет данных по объектам

Адаптив:
- на экранах планшета и мобильных устройств блок полностью скрывать;
- скрывать начиная примерно с ширины 1024px и меньше;
- на desktop блок должен оставаться справа после всех пунктов меню;
- желательно добавить margin-left: auto перед блоком, чтобы он прижимался к правой части меню.

Пример визуального поведения:
- 5 секунд отображается:
  «Всего в продаже 37 объектов»
- затем за 700ms эта строка плавно уезжает вверх;
- снизу появляется:
  «Последний объект добавлен 3 дня назад»
- ещё через 5 секунд анимация повторяется обратно.
